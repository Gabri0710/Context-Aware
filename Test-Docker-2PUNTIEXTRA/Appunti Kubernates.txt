Appunti kubernates:
- Si usa minikube (per usare kubernates)
- Si creano i DockerFile con le istruzioni per la creazione di ogni immagine, per ogni componente
- Si creano i deployment.yaml (definizioni dei vari container kubernates da creare, con riferimento a immagine da utilizzare e porta in ascolto nel container) per ogni componente
- Si creano i service.yaml (servizi esposti, permettono di accedere ai vari deployment) per ogni componente
- Nei service e i deployment vengono indicate porte in ascolto sul pc locale e targetport (porte in ascolto sul container) alla quale reindirizzare la richiesta. Il service è quello che si espone e che è in ascolto sulla porta specificata (port) e inoltra alla targetPort specificata la richiesta (targetPort sul service e port nel deployment devono essere uguali)
- una volta creato tutto:
- minikube start
- per ogni componente, nelle rispettive cartelle:
	- docker build -t nome_componente_image:tag .
	- docker tag nome_componente_image:tag nome_dockerhub/nome_componente_image:tag
	- docker login
	- docker push nome_dockerhub/nome_componente_image:tag
- andare nella cartella yaml:
	- kubectl apply -f database-deployment.yaml
	- kubectl apply -f database-service.yaml
	- kubectl apply -f backend-deployment.yaml
	- kubectl apply -f backend-service.yaml
	- kubectl apply -f frontend-deployment.yaml
	- kubectl apply -f frontend-service.yaml
- in dei cmd aperti con diritti di amministratore:
	- kubectl get pods 	->	mi dà la lista dei pods, aspettare che diventino running (rimandare il comando)
	- kubectl get service	->	mi dà la lista dei service
	- kubectl logs -l app=backend -> log dei backend (se può essere utile)
	- kubectl logs -l app=frontend -> log dei frontend (se può essere utile)
	- kubectl logs -l app=database -> log dei database (se può essere utile)

- per eseguire, su cmd aperti con diritti amministratore:
	- minikube tunnel		->	tunnel per accedere dall'esterno ai deployment tramite service (ci dà un ip locale)
	- minikube dashboard		->	apre la dashboard con informazioni sui pod ecc. Andare su service e cliccare sul frontend
